# 2026-06-22 — feature/domain-and-db

**What was built:** Created the three-project solution (Core, Infrastructure, Api), the domain entities (Shipment, ShipmentStatusHistory), the ShipmentStatus enum, IShipmentRepository interface, AppDbContext with relationship/index configuration, ShipmentRepository, and the InitialCreate EF Core migration.

**Key decisions:** Design package goes in the Api (startup) project, not Infrastructure; TrackingNumber uses HasMaxLength(50) before the unique index because SQL Server cannot index nvarchar(max); Update() is synchronous (EF tracks the entity immediately without hitting the DB) while all reads and SaveChangesAsync are async.

**Package split:** EntityFrameworkCore.SqlServer in Infrastructure (the provider belongs where the DbContext lives); EntityFrameworkCore.Design in Api (dotnet ef tooling discovers the startup project, not the migrations project).

**Blocker to resolve:** LocalDB is not installed on this machine — the migration files were generated successfully but `dotnet ef database update` cannot connect. Install SQL Server Express LocalDB (SqlLocalDB.msi) then run `dotnet ef database update -s ShipmentTracking.Api`.

## feature/shipment-crud

**What was built:** Added DTOs (CreateShipmentRequest, UpdateStatusRequest, ShipmentResponse, StatusHistoryResponse), InvalidStatusTransitionException, IShipmentService + ShipmentService (tracking-number generation + forward-only transition validation via a static dictionary), ApiKeyAuthFilter (fail-closed), and ShipmentsController with POST/GET/PUT endpoints returning 201/200/404/400.

**Key decisions:** Business rules (tracking-number uniqueness retry loop, AllowedTransitions dictionary, InvalidStatusTransitionException) all live in ShipmentService — the controller contains zero business logic and only translates null→404 and the domain exception→400. ApiKeyAuthFilter resolves IConfiguration from DI at request time, so no second DI registration is needed; fail-closed means missing config → 401 (not an accidental open door).

**Packages:** Zero new NuGet packages added. JsonStringEnumConverter chained onto AddControllers().AddJsonOptions() (not ConfigureHttpJsonOptions, which is minimal-API only) so enums serialize/deserialize as strings in both directions.

**Secret management:** `dotnet user-secrets` stores the dev API key in the OS user profile (not the repo). The UserSecretsId GUID in the csproj is safe to commit; the key itself never appears in any tracked file.
