# 2026-06-28

## Milestone 4.1 — Server-side RAG client (round-trip proof, no UI)

**What was built:** Added a server-side HTTP call from the .NET API to the deployed RAG service at `https://dethx-ragtest.hf.space/ask`. The endpoint `POST /api/assistant/ask` is public/anonymous; it forwards the caller's question to the RAG service and returns the grounded `answer` plus `sources` (each source has `document`, `score`, and `text`). The RAG API key stays server-side at all times.

**Key decisions:**
- Mirrored the existing JWT pattern exactly: `RagSettings` POCO bound via `GetSection("Rag").Get<RagSettings>()`, startup guard throws a helpful message if the key is missing, `Rag:ApiKey` comes from user-secrets locally / `Rag__ApiKey` env var on Render.
- Used a typed `HttpClient` (`AddHttpClient<IRagClient, RagClient>`) with base address and `Authorization: Bearer` header centralised in `Program.cs` — `RagClient` never touches the key directly. 100-second timeout for cold HF Spaces.
- Non-success outcomes (401, 429, timeout, 5xx) map to `RagClientException` with a caller-safe message and a `SuggestedStatusCode`; the controller maps that to 502/503, never a raw 500.
- `sources` shape confirmed from a real response: each element is `{ source, score, text }` — modeled as a typed `Source` DTO with `[JsonPropertyName]` mapping.

**Files added/changed:** `Core/Configuration/RagSettings.cs`, `Core/DTOs/AskRequest.cs`, `Core/DTOs/AskResponse.cs`, `Core/DTOs/Source.cs`, `Core/Interfaces/IRagClient.cs`, `Infrastructure/Exceptions/RagClientException.cs`, `Infrastructure/Rag/RagClient.cs`, `Api/Controllers/AssistantController.cs`, `Api/Program.cs` (edited), `Api/appsettings.json` (edited — `Rag:BaseUrl` added).

**To verify:** run `dotnet user-secrets set "Rag:ApiKey" "<key>"`, then `dotnet run --project ShipmentTracking.Api` and POST `{ "question": "what does CUS-HOLD mean?" }` to `/api/assistant/ask` — expect a 200 with `answer` and populated `sources`.
